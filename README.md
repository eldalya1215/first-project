## 3. 예외 처리 설계 (Exception Handling)

* **입력 정보 부족 및 부위 미입력**: 임의 생성을 차단하고, 내장된 퓨샷 템플릿을 통해 부족한 데이터를 역으로 되묻는(**Re-prompting**) 인터랙션을 수행합니다.
* **고위험 운동 강제 요청**: 사용자가 위험 동작을 직접 요구하더라도 시스템 거부권을 발동하고 부상 리스크를 경고한 뒤, 안전 대안 동작으로 **동적 스왑(Dynamic Swap)**합니다.
* **의료 판단 및 면책 제어**: 모든 출력의 ②오프닝 및 ⑥주의사항에 **"본 운동은 의학적 진단을 대체할 수 없으며 통증 시 즉시 중단"**하라는 경고 문구를 고정 삽입합니다.

## 4. 디자인 디시전 (Design Decision)

### ❓ GPT-5.5 코어 엔진 최종 선정 이유
* **GPT-5.5 (👑 4.92 / 5.0)**: 10턴 이상의 장기 테스트에서 포맷 미준수율 0%, 필터링 실패율 0% 기록. 대화 중간에 조건을 급선회("허리 ➡️ 무릎")해도 문맥 혼선이 없는 압도적 통제력 입증.
* **Claude 4 Sonnet (4.60 / 5.0)**: 톤앤매너는 자연스러우나, 장기 세션 누적 시 면책 고지 문구를 간헐적으로 누락함.
* **Gemini 2.5 Pro (4.28 / 5.0)**: 연산 속도는 가장 빠르나, 롱컨텍스트 환경에서 지정된 7단계 마크다운 구조 기호를 깨뜨리는 현상 발생.

### ❓ 프롬프트 아키텍처 버전 관리 (v1 ➡️ v2)
* **v1 (기본형)**: 서술형 단일 텍스트 배치 ➡️ 위험 동작 요구 시 환각 발생 및 출력 형식 불일치.
* **v2 (모듈형)**: 역할/제약/포맷/예외 분리 ➡️ **정형 포맷 이행률 100% 달성**, 선제적 거부권 발동 체계 정착.
